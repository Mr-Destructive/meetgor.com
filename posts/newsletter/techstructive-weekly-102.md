## Watched

- [The Primeagen on “I read the code”](https://youtu.be/k13q6ecZLrw)
  
  - This is solid stuff. I read the code. Its feeling like a ancient thing that used to be a myth. Reading code is ok, not like hard and fast rule to necessarily to be a developer. But when you want to prove a thing, and you have read the code, you have some confidence that no other metric or number can give you.
  - Vibe coding is fine, you don’t have to read the code always especially when performance or quality is not important, if getting the work done is important, which I agree are many things in the world, there vibe coding and blind developing a product or a software is fine. But not always.

<!--THE END-->

- [Computerphile: Why are tokens expensive](https://youtu.be/-0HRzXk8vlk)
  
  - This was a eye opening and a great explanation of how you are billed for each message. Highly recommend people to check and understand to open a new chat if you have been cramping on the same chat for ages.
  - Not only that becomes less expensive but makes the model a bit more clear on the things, otherwise its like finding a needle in a sand. Context bloat is a thing and so is the model’s ability to understand so many things at once.
  - Lesson from this, start a new chat if llm is starting to deteriorate and hallucinating. Maybe even before that. I know its tempting to keep the conversation going, the model knows. a lot of things, you feel a resistance to open a new chat and provide all the details again. but that might help you solve the problem more effectively with a fresh perspective otherwise the model will create a mess of the code.
