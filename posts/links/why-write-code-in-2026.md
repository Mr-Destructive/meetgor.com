## Commentary

- Doug Turnbull Blog: Why write code in 2026
- In the era where people defend with “I read the code”, this blog post slows down and answers a deeper question. Why write when nobody is even reading the code.
- Writing code has and will continue to give me clarity along the way. But AI has made all of that clarity to be required upfront which is a bit taxing on the brain.
- Like imagine starting to work on.a feature, you start with the boilerplate code and then start to write something, then you realize oh, that is not right, we need this too, we won’t require that I think. And you get the idea, the clarity comes on the way and not staring at the agent after leaving it with a one line description of the task.
